## 执行流程

```
1. Agent 领取任务 (读取 TODO.md)
2. 创建 GitHub Issue 分支
3. 开发实现
4. 单元测试
5. Code Review (architect)
6. 合并到 main
7. 更新 TODO.md 状态
8. 记录到 agent-log.txt
```

## 审核标准

### Code Review 检查清单
- [ ] 代码风格符合 Rust/TypeScript 规范
- [ ] 单元测试覆盖率 > 80%
- [ ] 无安全漏洞 (串口权限、网络连接)
- [ ] 错误处理完善
- [ ] 日志记录完整

### 功能验收检查清单
- [ ] 功能符合 SPEC.md 规格
- [ ] 性能达标 (响应时间 < 100ms)
- [ ] 跨平台测试通过
- [ ] 文档已更新
